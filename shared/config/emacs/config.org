* Org

This section configures Org as a small personal productivity system.  The basic
loop is: capture quickly into the inbox, clarify the inbox into tasks and
projects, work from the dashboard, and review regularly.  Org-roam holds notes
that should outlive an individual task.  Emacs Calendar and Diary supply dated
appointments to the agenda.

** Quick start

The three commands worth learning first are:

- =C-c c= captures something without interrupting the current task.
- =C-c A= opens the daily dashboard.
- =C-c a= opens Org's full agenda dispatcher, including the weekly review.

A simple daily routine is:

1. Open the dashboard with =C-c A= and choose what to work on.
2. Capture new work with =C-c c t= instead of reorganizing files immediately.
3. Mark the current action =DOING= and optionally clock into it with =C-c o i=.
4. During an inbox pass, refile captured entries with =C-c C-w=.
5. Mark completed work =DONE=.  Finishing the clocked task automatically clocks
   out.

** Files and storage

The Org root is =~/org/= by default.  Set the =ORG_DIRECTORY= environment
variable before Emacs starts to use a different directory.  Missing core files
and directories are created automatically.

- =inbox.org= is the landing place for unprocessed tasks, ideas, links, and
  journal entries.
- =tasks.org= contains standalone actions, waiting items, someday items, and
  habits.
- =projects.org= contains multi-step outcomes and meeting notes.
- =roam/= contains durable notes, people, references, project notes, and daily
  notes.
- =archive/= receives completed subtrees archived with =C-c C-x C-a=.
- =diary.org= stores Emacs Calendar appointments in the synchronized Org directory.

The agenda reads the inbox, task, and project files plus the Emacs diary.  The Org
directory is synchronized every 15 minutes by the =org-webdav-sync.timer=
systemd user unit.  Emacs queues that service asynchronously on startup and
shutdown, so a slow network cannot delay either operation.

** Capturing

Press =C-c c= and then choose a template:

| Key | Template       | Destination and purpose                         |
|-----+----------------+-------------------------------------------------|
| =t= | Inbox task     | A plain =TODO= in =inbox.org=                   |
| =s= | Scheduled task | An inbox task with a prompted date and time     |
| =i= | Idea           | An inbox entry tagged =idea=                    |
| =l= | Link           | A stored link back to the current context       |
| =m= | Meeting        | A dated meeting record in =projects.org=        |
| =j= | Journal        | A dated journal entry in =inbox.org=            |
| =h= | Habit          | A repeating task under =tasks.org= / Habits     |
| =p= | Project        | A project outcome and first next action         |

*** What to fill in the capture buffer

The first line is the headline.  Put a short, specific name immediately after
the TODO keyword or the leading =*=.  This is what appears in searches and the
agenda, so it should make sense without reading the rest of the entry.  Put
supporting details, links, and longer thoughts in the body below the property
drawer.

For example, an idea capture initially looks like this:

#+begin_example
*  :idea:
:PROPERTIES:
:CREATED: [2026-07-11 Sat 22:11]
:END:
#+end_example

Fill it in like this:

#+begin_example
* Add lag simulation to the multiplayer test harness :idea:
:PROPERTIES:
:CREATED: [2026-07-11 Sat 22:11]
:END:

It would be useful to control latency, jitter, and packet loss independently.
Try it first on the local WebSocket transport.
#+end_example

In other words, do not leave the headline as only =* :idea:=.  In the screenshot
above, =I think it would be cool, if a lag ...= could become a concise headline
such as =Simulate network lag in local tests=, followed by the full thought in
the body.

What each template expects:

- Inbox task (=t=): replace the cursor with an actionable phrase, such as
  =Email Dara the revised proposal=.  Add context or a checklist in the body.
- Scheduled task (=s=): enter the action first, then choose when Emacs prompts
  for a date.  Use a schedule for when work should begin, not merely when it is
  due; add a deadline later with =C-c C-d= if needed.
- Idea (=i=): write a short name for the idea on the headline and explain the
  motivation, possible implementation, or next experiment in the body.  An idea
  does not need a TODO state until it becomes committed work.
- Link (=l=): name why the saved page or location matters.  The stored link and
  captured context are inserted automatically; add your own note below them.
- Meeting (=m=): enter the meeting name at the prompt, write notes under
  =Notes=, and put concrete follow-ups below =NEXT Follow-ups=.  This template
  starts a clock while capturing.
- Journal (=j=): the timestamp is supplied automatically.  Write the journal
  subject at the cursor, then continue with any detail beneath it.
- Habit (=h=): enter the repeatable behavior, such as =Review Spanish cards=,
  and choose the first scheduled date.  In the date prompt, add a repeater such
  as =+1d= for daily or =+1w= for weekly.
- Project (=p=): enter an outcome rather than a vague topic, for example
  =Publish the portfolio website=.  At the =NEXT= child, enter the first visible
  physical action, such as =Choose three projects to feature=.

The =PROPERTIES= drawer is metadata.  Normally leave =CREATED= and the drawer
markers unchanged.  Move below =:END:= before adding body text; text placed
inside the drawer must be valid =:NAME: value= metadata and is not ordinary
notes.

The capture window's header shows the available exits.  Use =C-c C-c= when the
entry is ready, =C-c C-w= to finish and immediately choose a different filing
location, or =C-c C-k= to discard the capture.  You can edit and move around the
temporary buffer normally before choosing one of those commands.

Use =C-c l= first when you want to remember the current file, web page, email,
or other linkable context; then =C-c c l= captures it.  Finish a capture with
=C-c C-c= or cancel it with =C-c C-k=.

The inbox is deliberately temporary.  When processing it, delete irrelevant
items, do very small actions, or use =C-c C-w= to refile an entry into the
appropriate heading in =tasks.org= or =projects.org=.

** Task states, projects, and tags

Tasks move through this sequence:

#+begin_example
TODO -> NEXT -> DOING -> WAIT -> SOMEDAY -> DONE / CANCELLED
#+end_example

- =TODO= means the work exists but has not been clarified as an immediate next
  action.
- =NEXT= is ready to do.
- =DOING= is actively in progress.
- =WAIT= is blocked on another person or event.  Changing to this state prompts
  for a note.
- =SOMEDAY= is intentionally deferred and also prompts for a note.
- =DONE= and =CANCELLED= are terminal states.  State changes and completion
  times are recorded in a drawer.

Cycle states with =C-c C-t=.  Schedule work with =C-c C-s=, add a deadline with
=C-c C-d=, and set tags with =C-c C-q=.  Available context tags are =@home=,
=@work=, =@computer=, and =@errand=; descriptive tags include =project=, =idea=,
and =meeting=.

A project is an unfinished heading tagged =project=.  Give every active project
at least one child action in =NEXT= or =DOING=; otherwise Org reports it as a
stuck project during the weekly review.  Task and checkbox dependencies are
enforced, so parent work cannot be completed while required children remain
open.

** Agenda and reviews

=C-c A= opens the custom daily dashboard.  It shows today's calendar, work in
progress, next actions, overdue work, waiting items, and active projects.

=C-c a= opens the dispatcher.  The custom views are:

- =d= -- the same daily dashboard.
- =w= -- a seven-day weekly review with stuck projects, waiting tasks, someday
  tasks, and all active projects.
- =p= -- active projects only.

In an agenda, use =RET= to visit an item, =t= to change its TODO state, =s= to
schedule it, =d= to set a deadline, and =g= to refresh the view.  The normal Org
agenda help key =?= shows all commands available in the current view.

A useful weekly review is to empty =inbox.org=, inspect every stuck project,
follow up or reschedule =WAIT= items, reconsider =SOMEDAY= items, and make sure
each active project has a concrete =NEXT= action.

** Time tracking

- =C-c o i= clocks into the heading at point.
- =C-c o o= clocks out.
- =C-c o g= jumps to the currently clocked task.

Clock history persists across Emacs restarts, and clocking into a previous task
resumes it.  Moving the currently clocked task to =DONE= or =CANCELLED= clocks
out automatically.  Use Org's clock table facilities when a note or project
needs a time report.

** Org-roam notes

Org-roam is for connected knowledge rather than actionable work:

- =C-c o f= finds or creates a note.
- =C-c o n= inserts a link to a note.
- =C-c o b= toggles the backlinks buffer.
- =C-c o d= opens today's daily note.
- =C-c o D= captures directly into today's daily note.

When creating a note, choose =n= for a general note, =p= for a person, =r= for a
reference, or =P= for a project note.  Project notes contain supporting context;
the actionable project itself still belongs in =projects.org=.  Daily notes
provide a morning plan, chronological log, meeting area, and end-of-day review.

** Calendar and diary

Press =M-x calendar= to open Emacs Calendar.  Press =i d= on a date to add a
diary appointment.  The diary lives at =~/org/diary.org=, so the existing WebDAV
service synchronizes it with the rest of the Org directory.  Diary entries are
shown in the normal Org agenda and startup dashboard.

** Writing Org documents

In Org buffers, long lines are filled at 80 columns.  Type =<el=, =<py=, or
=<sh= followed by =TAB= to insert an Emacs Lisp, Python, or shell source block.
Run the block with =C-c C-c=; inline images are refreshed afterward.  Links open
with =RET=, and non-Org files use the desktop's normal application.

This file is the source of truth.  After changing the Lisp block below, tangle

(defun tsp/org-git-sync (&optional quiet)
  "Commit, rebase, and push the Org repository.
When QUIET is non-nil, only report failures.  Conflicts and network errors are
left untouched for manual recovery."
  (interactive)
  (when (process-live-p tsp/org-git-sync-process)
    (delete-process tsp/org-git-sync-process)
    (setq tsp/org-git-sync-process nil))
  (unless tsp/org-git-sync-running
    (let ((tsp/org-git-sync-running t))
      (tsp/org-save-buffers)
      (if (not (file-directory-p (expand-file-name ".git" tsp/org-directory)))
          (unless quiet
            (message "Org Git sync skipped: %s is not a repository"
                     tsp/org-directory))
        (let* ((status (tsp/org-git-run "status" "--porcelain"))
               (dirty (not (string-empty-p (cdr status))))
               failure)
          (when (/= (car status) 0)
            (setq failure (cdr status)))
          (when (and dirty (not failure))
            (pcase-let ((`(,add-status . ,add-output)
                         (tsp/org-git-run "add" "--all")))
              (if (/= add-status 0)
                  (setq failure add-output)
                (pcase-let ((`(,commit-status . ,commit-output)
                             (tsp/org-git-run
                              "commit" "-m"
                              (format-time-string "Org sync: %Y-%m-%d %H:%M"))))
                  (when (/= commit-status 0)
                    (setq failure commit-output))))))
          (unless failure
            (pcase-let ((`(,pull-status . ,pull-output)
                         (tsp/org-git-run "pull" "--rebase")))
              (when (/= pull-status 0)
                (setq failure pull-output))))
          (unless failure
            (pcase-let ((`(,push-status . ,push-output)
                         (tsp/org-git-run "push")))
              (when (/= push-status 0)
                (setq failure push-output))))
          (if failure
              (display-warning
               'tsp-org
               (format "Org Git sync failed; files were preserved:\n%s" failure)
               :warning)
            (unless quiet
              (message "Org Git sync complete"))))))))

(defun tsp/org-git-sync-async ()
  "Synchronize the Org repository without blocking Emacs."
  (interactive)
  (tsp/org-save-buffers)
  (if (not (file-directory-p (expand-file-name ".git" tsp/org-directory)))
      (message "Org Git sync skipped: %s is not a repository" tsp/org-directory)
    (unless (process-live-p tsp/org-git-sync-process)
      (let* ((default-directory tsp/org-directory)
             (buffer (get-buffer-create " *Org Git Sync*"))
             (commit-message
              (shell-quote-argument
               (format-time-string "Org sync: %Y-%m-%d %H:%M")))
             (command
              (string-join
               (list "git add --all"
                     (format (concat "if ! git diff --cached --quiet; then "
                                     "git commit -m %s; fi")
                             commit-message)
                     "git pull --rebase"
                     "git push")
               " && ")))
        (with-current-buffer buffer
          (erase-buffer))
        (setq tsp/org-git-sync-process
              (make-process
               :name "org-git-sync"
               :buffer buffer
               :command (list "/bin/bash" "-lc" command)
               :noquery t
               :sentinel
               (lambda (process _event)
                 (when (memq (process-status process) '(exit signal))
                   (setq tsp/org-git-sync-process nil)
                   (if (= (process-exit-status process) 0)
                       (kill-buffer (process-buffer process))
                     (display-warning
                      'tsp-org
                      (format
                       "Background Org Git sync failed; see %s"
                       (buffer-name (process-buffer process)))
                      :warning))))))))))

(defun tsp/telega-khmer-string-p (string)
  "Return non-nil when STRING contains Khmer characters."
  (string-match-p "[ក-៿]" string))

(defun tsp/telega-rendered-string-width (string &optional from to)
  "Measure STRING using its rendered glyph width when it contains Khmer."
  (if (and (display-graphic-p)
           (tsp/telega-khmer-string-p
            (if (or from to) (substring string from to) string)))
      (telega-window-string-width string from to)
    (string-width string from to)))

(defun tsp/telega-glyph-boundaries (string)
  "Return a vector of safe character boundaries in STRING.
Avoid `string-glyph-split', which can abort Emacs 30.2 while Telega renders
some chat-preview strings."
  (let ((position 0)
        (boundaries (list 0)))
    (while (< position (length string))
      (setq position (1+ position))
      (push position boundaries))
    (vconcat (nreverse boundaries))))

(defun tsp/telega-fit-prefix-end (string boundaries max-width limit)
  "Return the longest grapheme-safe prefix of STRING within MAX-WIDTH.
Do not return a position beyond LIMIT."
  (let ((low 0)
        (high (1- (length boundaries)))
        (best 0))
    (while (<= low high)
      (let* ((middle (/ (+ low high) 2))
             (end (aref boundaries middle)))
        (if (or (> end limit)
                (> (tsp/telega-rendered-string-width string 0 end) max-width))
            (setq high (1- middle))
          (setq best end
                low (1+ middle)))))
    best))

(defun tsp/telega-fit-suffix-start (string boundaries max-width)
  "Return the earliest grapheme-safe suffix fitting within MAX-WIDTH."
  (let ((low 0)
        (high (1- (length boundaries)))
        (best (length string)))
    (while (<= low high)
      (let* ((middle (/ (+ low high) 2))
             (start (aref boundaries middle)))
        (if (<= (tsp/telega-rendered-string-width string start) max-width)
            (setq best start
                  high (1- middle))
          (setq low (1+ middle)))))
    best))

(defun tsp/telega-khmer-aware-eliding (original string properties)
  "Elide Khmer STRING by rendered grapheme width.
Call ORIGINAL for strings that do not contain Khmer."
  (if (not (tsp/telega-khmer-string-p string))
      (funcall original string properties)
    (let ((max-width (plist-get properties :max)))
      (if (or (not max-width)
              (<= (tsp/telega-rendered-string-width string) max-width))
          string
        (let* ((elide-string (or (plist-get properties :elide-string)
                                 telega-symbol-eliding))
               (elide-width (tsp/telega-rendered-string-width elide-string))
               (elide-position (or (plist-get properties :elide-position) 1))
               (available-width (max 0 (- max-width elide-width)))
               (trail-width (floor (* available-width
                                      (- 1 elide-position))))
               (boundaries (tsp/telega-glyph-boundaries string))
               (trail-start
                (tsp/telega-fit-suffix-start string boundaries trail-width))
               (actual-trail-width
                (tsp/telega-rendered-string-width string trail-start))
               (lead-width (max 0 (- available-width actual-trail-width)))
               (lead-end
                (tsp/telega-fit-prefix-end
                 string boundaries lead-width trail-start)))
          (when (< lead-end trail-start)
            (add-text-properties
             lead-end trail-start
             (list 'display elide-string
                   'rear-nonsticky '(display)
                   'face (plist-get properties :face))
             string))
          string)))))

(defun tsp/dashboard-diary-agenda ()
  "Return Diary appointments for the dashboard's seven-day agenda."
  (require 'diary-lib)
  (mapcar
   (lambda (entry)
     (let* ((date (car entry))
            (lines (split-string (nth 1 entry) "\n[[:space:]]*" t))
            (text (concat (car lines)
                          (when (cdr lines)
                            (concat "\n        "
                                    (string-join (cdr lines) "\n        ")))))
            (source (nth 3 entry))
            (marker (car source)))
       (propertize
        (format "%04d-%02d-%02d  %s"
                (nth 2 date) (car date) (nth 1 date) text)
        'dashboard-agenda-file (nth 1 source)
        'dashboard-agenda-loc (if (markerp marker) (marker-position marker) 1))))
   (diary-list-entries (calendar-current-date) 7 t)))

    (dashboard-insert-section

     items list-size 'agenda (dashboard-get-shortcut 'agenda)
     `(lambda (&rest _)
        (let ((file (get-text-property 0 'dashboard-agenda-file ,el))
              (point (get-text-property 0 'dashboard-agenda-loc ,el)))
          (find-file file)
          (goto-char point)))
     (format "%s" el))))

(defun tsp/dashboard-refresh-if-visible ()
  "Refresh the dashboard when it already exists."
  (when-let* ((buffer (get-buffer "*dashboard*")))

(defun tsp/dashboard-open ()
  "Open a freshly rendered dashboard, including after its buffer was killed."
  (interactive)
  (when-let ((buffer (get-buffer "*dashboard*")))
    ;; Do not use `with-current-buffer' here: when this command is invoked
    ;; from the dashboard, killing BUFFER would leave the unwind code trying
    ;; to restore a current buffer that no longer exists.
    (when (provided-mode-derived-p (buffer-local-value 'major-mode buffer)
                                   'dashboard-mode)
      (kill-buffer buffer)))
  (dashboard-open))

  "Open the dashboard after other startup buffer changes have finished."
  (when (and (not noninteractive) (not (daemonp)))
    (tsp/dashboard-open)))

  (defun tsp/emms-refresh-library (&optional quiet)
    "Rescan the music directory and refresh the EMMS browser.
When QUIET is non-nil, avoid status messages for automatic refreshes."
    (interactive)
    (let ((library (expand-file-name emms-source-file-default-directory)))
      (if (not (file-directory-p library))
          (unless quiet
            (message "EMMS library is not mounted: %s" library))
        (unless quiet
          (message "Refreshing EMMS library from %s..." library))
        ;; Populate the cache without adding the entire library to the user's queue.
        (with-temp-buffer
          (setq emms-playlist-buffer-p t
                emms-playlist-insert-track-function
                #'emms-playlist-mode-insert-track)
          (let ((emms-playlist-buffer (current-buffer)))
            (emms-add-directory-tree library)))
        ;; Older cache files may contain the printer's truncation marker,
        ;; which hides properties appearing after it (notably track titles).
        (maphash (lambda (path track)
                   (setcdr track (delq (intern "...") (cdr track)))
                   (when (and (emms-track-file-p track)
                              (file-in-directory-p path library))
                     (emms-info-native track)))
                 emms-cache-db)
        (emms-cache-sync t)
        (let (print-length print-level)
          (emms-cache-save))
        (when (buffer-live-p emms-browser-buffer)
          (with-current-buffer emms-browser-buffer
            (emms-browse-by emms-browser-default-browse-type)))
        (unless quiet
          (message "EMMS library refresh complete")))))

  (when (timerp tsp/emms-library-refresh-timer)
    (cancel-timer tsp/emms-library-refresh-timer))
  (setq tsp/emms-library-refresh-timer
        (run-with-idle-timer 60 600 #'tsp/emms-refresh-library t))

  (emms-playing-time-mode 1))
